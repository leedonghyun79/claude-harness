# 보안

## 위협 모델

| 위협 | 리스크 | 대응 |
|------|--------|------|
| XSS | 중간 | React 기본 이스케이프; `dangerouslySetInnerHTML` 사용 금지 |
| IDOR | 높음 | 백엔드에서 리소스 소유권 검증 |
| 인증 우회 | 높음 | 미들웨어/세션 기반 인증, if문 단독 보호 금지 |
| 시크릿 노출 | 높음 | 클라이언트 코드에 API 키 포함 금지 |
| SQL/NoSQL 인젝션 | 높음 | ORM/쿼리 빌더 사용, raw query 지양 |
| 프롬프트 인젝션 | 중간 | 사용자 입력과 시스템 프롬프트 분리 |
| 의존성 취약점 | 중간 | CI에서 `npm audit` 실행 |
| 과도한 과금 | 중간 | 외부 API usage limit + 비용 알림 설정 |

---

## 필수 규칙

- `dangerouslySetInnerHTML` 사용 금지 (명시적 살균 없이는)
- CI에서 `npm audit --audit-level=high`
- 클라이언트 코드에 시크릿 포함 금지
- 환경 변수는 `.env.local`에만, 절대 커밋 금지
- JWT는 `httpOnly` 쿠키에 저장 (localStorage 금지)
- 백엔드에서도 인증/권한 검증 필수 (프론트만 막으면 백엔드는 열려있음)

## 보안 체크리스트

### 1. 일반 보안 점검

- [ ] "기능이 작동하는가" + "작동하지 말아야 할 때도 작동하는가"를 모두 테스트하는가?
- [ ] 내 계정으로 남의 데이터 조회되는가? (IDOR) / URL 파라미터 바꿔도 막히는가?
- [ ] 로그아웃 후 이전 URL 접근 막히는가?
- [ ] 보호되지 않은 라우트가 있는가?

### 2. 인증 및 접근 제어

**관리자 페이지**
- [ ] `/admin`, `/dashboard`, `/internal` 등이 인증 없이 접근 불가한가?
- [ ] 관리자 페이지를 "숨김"이 아닌 "인증"으로 보호하는가?
- [ ] if문 하나가 아닌 미들웨어/세션 기반 인증을 사용하는가?

**비밀번호 재설정**
- [ ] 재설정 토큰은 1회용인가?
- [ ] 토큰 만료 시간이 15분 이내인가?
- [ ] 성공/실패 응답 메시지가 동일한가? ("이메일이 없습니다" 노출 금지)

**소셜 로그인(OAuth)**
- [ ] OAuth state 파라미터를 검증하는가?
- [ ] 콜백 URL 화이트리스트가 있는가?
- [ ] 토큰 만료를 처리하는가?
- [ ] 로그아웃 시 토큰을 무효화하는가?

**일반**
- [ ] Brute Force 방어(로그인 시도 횟수 제한)가 있는가?
- [ ] 세션 타임아웃, 동시 로그인 제한이 설정되어 있는가?
- [ ] 백엔드에서도 인증/권한 검증을 하는가?

### 3. 키 및 시크릿 관리

- [ ] `service_role key`, `secret key`, `admin key`가 서버에서만 접근 가능한가?
- [ ] `NEXT_PUBLIC_` 환경변수에 시크릿이 노출되지 않았는가?
- [ ] Git 히스토리에 시크릿이 포함된 적이 있는가?
- [ ] `.env` 파일이 `.gitignore`에 포함되어 있는가?
- [ ] 빌드된 JS 번들에서 시크릿이 포함되지 않았는가?

### 4. Supabase / Next.js 특화

**Supabase**
- [ ] RLS(Row Level Security)가 켜져 있는가?
- [ ] anon key로 전체 테이블 조회가 되는가? (되면 이미 뚫린 것)
- [ ] RLS 정책을 실제로 테스트해봤는가?

**Next.js**
- [ ] `getServerSideProps`에서 유저 인증을 검증하는가?
- [ ] 클라이언트에서 직접 Supabase 쿼리를 하지 않는가?
- [ ] 모든 API route에 인증이 붙어있는가?

### 5. 프롬프트 인젝션 방어 (AI 기능 사용 시)

- [ ] 시스템 프롬프트에 API 키, 비즈니스 로직 등 민감 정보가 없는가?
- [ ] 사용자 입력과 시스템 프롬프트를 분리하는가?
- [ ] AI 엔드포인트에 Rate Limiting이 있는가?
- [ ] 댓글/리뷰 기능에 XSS 방어(HTML 이스케이핑)가 적용되어 있는가?

### 6. 로깅 및 데이터 보호

- [ ] 프로덕션에 `console.log`가 남아있지 않은가?
- [ ] 로그에 비밀번호, 이메일, 카드번호가 기록되지 않는가?
- [ ] 로그에 JWT 토큰 전체, API 키가 기록되지 않는가?
- [ ] JWT에 비밀번호 / 신용카드 / 주민번호 / 전체 유저 객체가 없는가?
- [ ] JWT secret이 충분히 강력하고 만료 시간이 적절한가?

### 7. 과금 및 비용 보안

- [ ] AWS, OpenAI, Stripe 등에 월별 비용 알림이 설정되어 있는가?
- [ ] Stripe 테스트 키와 프로덕션 키가 확실히 분리되어 있는가?
- [ ] 외부 API 키에 usage limit이 설정되어 있는가?

### 8. 코드 위생

- [ ] MVP 단계에서도 최소 보안(인증, 키 분리, HTTPS)을 적용하는가?
- [ ] "임시로 해둔" 코드(TODO, HACK, TEMP)가 프로덕션에 남아있지 않은가?
- [ ] 의존성 취약점 스캔(`npm audit`, Snyk 등)을 하는가?
- [ ] 불필요한 엔드포인트, 미사용 기능이 비활성화되어 있는가?

### 9. 법률 및 컴플라이언스

- [ ] 개인정보 처리방침을 공개하고 있는가?
- [ ] 개인정보 수집 전 사용자 동의를 받고 있는가?
- [ ] 민감 정보(주민번호, 카드번호 등)가 암호화되어 저장되는가?
- [ ] 보유 기간이 지난 개인정보를 즉시 파기하는가?
